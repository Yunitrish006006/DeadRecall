# 銅魁儡使用指南

本文件說明 DeadRecall v2.2.1 銅魁儡與銅板手的玩家操作方式。技術規格請看 `openspec/specs/copper-golem/spec.md`。

## v2.2.1 重點

- 採集掃描每 tick 最多檢查 512 個候選方塊。
- 採集成功後會從上次掃描位置繼續找下一個目標，不再每次從工作區最上層重頭掃。
- 採集站位改用銅魁儡本身碰撞箱判定，不再硬性要求玩家兩格高空間。
- 銅魁儡可用銅錠右鍵修復。

## 核心概念

每一隻銅魁儡都有自己的設定資料，銅板手只是用來選取與管理指定銅魁儡。

銅魁儡目前有兩種模式：

| 模式 | 用途 | 來源銅箱用途 | 其他容器用途 |
| --- | --- | --- | --- |
| 分類 | 從來源銅箱拿物品，分到已綁定的目標容器 | 待分類物品來源 | 分類目的地 |
| 採集 | 在指定作業區域採集方塊，放回來源銅箱 | 採集 Home / 存放倉庫 | 不綁定一般容器 |

來源銅箱每隻銅魁儡只能綁一個。分類模式與採集模式共用同一個來源銅箱，但用途不同。

## 基本操作表

| 操作 | 條件 | 效果 |
| --- | --- | --- |
| 右鍵銅魁儡 | 手持銅錠，銅魁儡受傷 | 消耗 1 個銅錠，修復 4 點生命值 |
| Shift+右鍵銅魁儡 | 手持銅板手 | 將板手綁定到該銅魁儡，並開啟管理 GUI |
| 右鍵銅箱 | 板手已綁定銅魁儡 | 設定或替換來源銅箱 |
| 左鍵來源銅箱 | 板手已綁定銅魁儡 | 解除來源銅箱 |
| 右鍵一般容器 | 分類模式 | 加入分類目的地 |
| 左鍵已綁定的一般容器 | 分類模式 | 從分類目的地清單移除 |
| 右鍵普通方塊 | 採集模式 | 設定採集 Corner A |
| Shift+右鍵普通方塊 | 採集模式 | 設定採集 Corner B |
| 左鍵普通方塊 | 採集模式 | 新增或移除該 Block ID 的手動採集目標 |
| 手持已綁定板手 | 與銅魁儡同維度 | 顯示目前模式的視覺化資訊 |

跨維度不能綁定。銅魁儡與目標容器或作業區方塊必須在同一維度。

## 管理 GUI

用銅板手 Shift+右鍵銅魁儡會開啟管理 GUI。

GUI 主要用來管理：

| 區塊 | 用途 |
| --- | --- |
| 模式 | 在分類與採集之間切換 |
| 運作 / 停止 | 控制銅魁儡是否開始工作 |
| 燃料槽 | 放入銅魁儡工作需要的燃料 |
| 來源銅箱 | 在燃料槽旁以 icon 顯示目前綁定的來源銅箱 |
| 分類目的箱 | 分類模式下顯示綁定容器與分類快取 |
| 採集工具欄 | 採集模式下放入採集工具 |
| 採集倉庫 | 採集模式下暫存最多 16 個採集物 |
| Prompt | 在目標容器卡片或採集目標區塊內設定 LLM 判斷文字 |
| LLM | 設定 API URL、API Key、Model，並測試連線 |

來源銅箱只會在燃料槽旁顯示一個 icon。座標、方塊 ID、載入狀態與可用狀態請把滑鼠移到該 icon 上查看。

分類目的箱未選取時會以精簡卡片顯示，只顯示箱子本身與名稱。點選目的箱後，該卡片會展開並顯示 LLM 開關與接受 / 拒絕快取；只有該箱子的 LLM 開啟時才會顯示 Prompt 輸入框。

箱子分頁不會顯示固定的 LLM / Prompt 摘要卡片；分類清單與採集內容會直接從上方標題列下方開始。

燃料槽與採集工具欄的操作規則：

| 欄位 | 左鍵 | 右鍵 |
| --- | --- | --- |
| 燃料槽 | 從主手或背包自動放入燃料 | 取回燃料 |
| 採集工具欄 | 從主手或背包自動放入工具 | 取回工具 |
| 採集倉庫 | 無 | 取回採集物 |

切換模式前必須先停止銅魁儡。若銅魁儡手上有分類貨物、採集工具仍在欄位中、或採集倉庫還有物品，伺服器會拒絕切換。

## 分類模式

分類模式的流程：

1. 用銅板手 Shift+右鍵銅魁儡，開啟 GUI。
2. 確認模式是「分類」。
3. 右鍵一個銅箱，設定來源銅箱。
4. 右鍵一般容器，加入分類目的地。
5. 在 GUI 左鍵燃料槽，讓系統從主手或背包自動放入燃料。
6. 按「運作」。

銅魁儡會從來源銅箱拿待分類物品，每次最多拿 16 個相同 ItemStack，然後依序尋找可放入的分類目的地。

分類目的地判斷順序：

1. 如果該 item id 或 tag 在該容器的拒絕快取中，直接拒絕。
2. 如果容器或容器內的 DeadRecall 背包有相同 Item + Components 且可合併，可以放入。
3. 如果該 item id 或 tag 在該容器的接受快取中，且容器有空位，可以放入。
4. 如果容器 LLM 已啟用、Prompt 有填、且該物品沒有快取，會送出 LLM 判斷請求，本輪先跳過等待結果。
5. 如果所有目的地都找不到位置，會嘗試把物品放回來源銅箱。

分類快取是共用規則，不只給 LLM 使用。手動調整、LLM 結果、以及相同物品分類都會一起看這份快取。右鍵接受或拒絕區的物品圖示，可以把該項目移到另一邊。

分類 Prompt 只會在目前選取且 LLM 已開啟的目標容器卡片內顯示。先開啟該箱子的 LLM，再輸入 Prompt 並按儲存。

## 採集模式

採集模式的流程：

1. 用銅板手 Shift+右鍵銅魁儡，開啟 GUI。
2. 停止銅魁儡。
3. 切換到「採集」模式。
4. 右鍵銅箱，設定來源銅箱作為 Home。
5. 右鍵普通方塊設定 Corner A。
6. Shift+右鍵普通方塊設定 Corner B。
7. 左鍵想採集的方塊，加入手動採集目標。
8. 在 GUI 左鍵工具欄與燃料槽，讓系統從主手或背包自動放入工具與燃料。
9. 按「運作」。

採集作業區域由 Corner A 與 Corner B 組成。銅魁儡只會掃描已載入區塊，不會為了掃描而強制載入未載入區塊。

採集時會檢查：

| 檢查項目 | 說明 |
| --- | --- |
| 方塊是否安全 | 容器、來源銅箱、不可破壞方塊、流體、保護區會被拒絕 |
| 是否在作業區域內 | 超出區域不採集 |
| 工具是否有效 | 需要合法採集工具 |
| 掉落物是否能放入倉庫 | 採集倉庫最多 16 個，只收可合併的同類物品 |
| 燃料是否足夠 | 成功破壞方塊才消耗工作燃料 |

採集倉庫未滿 16 個且仍有可合併目標時，銅魁儡會繼續採集。倉庫滿了，或掃完整個作業區後找不到可繼續合併的目標，才會回來源銅箱存放。

一旦銅魁儡因找不到更多目標而開始回來源銅箱，就會持續完成存放，不會因採集倉庫未滿 16 個而反覆重新掃描或切換手上顯示物。

搜尋採集目標時，銅魁儡會從選取工作區域的最上層開始，由上往下找有效方塊，避免先挖下面造成上方資源懸空或漏掉。

大型作業區不會一次掃完；目前每 tick 最多檢查 512 個候選方塊。成功採集後會從上次掃描位置繼續找下一個目標，不會每次都重新從工作區最上層開始。

如果目標方塊因路徑失敗或長時間沒有接近而挖不到，銅魁儡會暫時跳過該方塊並搜尋下一個目標，不會消耗燃料或改動工具。

銅魁儡會嘗試站到目標方塊下方或周圍的可站位置，因此可以採集頭頂上方範圍內的方塊；目前上方採集範圍是以頭頂上一格開始的 3x3 區域，高度 2 格，不是頭頂延伸出去的所有方塊都能挖。

站位判定會使用銅魁儡本身的碰撞箱，不會要求像玩家一樣保留完整兩格高空間；只要腳下有穩固支撐且銅魁儡實際碰撞箱塞得進去，就可以被當作採集站位。

遇到地表層、草地方塊或側面被遮住的方塊時，銅魁儡會優先嘗試站到目標上方往下挖。向下採集範圍同樣是有限 3x3 區域，深度 2 格。

銅魁儡抵達目標後會依方塊硬度、工具挖掘速度與採集相關附魔累積破壞進度，效率約為同工具玩家的一半。畫面上會看到方塊裂痕與揮動工具，成功破壞後才會扣燃料、依耐久附魔扣工具耐久並把掉落物放入採集倉庫。

銅魁儡必須靠近到目標方塊旁邊才會開始破壞；一般水平採集距離約 2 格，上方與向下採集都只限有限範圍。

採集工具只會在真正挖掘時顯示在銅魁儡手上；搜尋或走向目標時不顯示工具。回倉庫存放時會顯示採集倉庫中的物品。這些手持物只是採集系統的視覺顯示，不能用空手右鍵從銅魁儡身上拿走；要取回工具或採集物必須停止後使用 GUI 欄位。

採集目標區會分成「接受目標」與「拒絕」。手動左鍵指定的方塊會顯示在接受目標中，Prompt 快取的允許 / 拒絕方塊與 Tag 也會顯示在對應欄位；右鍵點目標圖示可以移除該目標。

採集 Prompt 會顯示在採集目標區塊上方，和手動目標、Prompt 快取顯示在同一個區域。採集模式停止時，只要 LLM 已啟用、Prompt 與 API 可用、工作區兩個角點已完成且工具欄有工具，銅魁儡會低頻掃描工作區來預熱採集快取。工具等級不足、工具無法取得掉落、容器、安全拒絕方塊或不可破壞方塊不會送 LLM，也不會被寫進拒絕快取。

## LLM 設定

每隻銅魁儡共用一組 LLM API 設定：

| 欄位 | 內容 |
| --- | --- |
| API URL | OpenAI 相容的 chat completions endpoint |
| API Key | 供應商需要的 key，本地服務可填任意值或服務要求的值 |
| Model | 供應商模型名稱 |

分類模式中，每個分類目的箱有自己的 LLM 開關與 Prompt。Prompt 用來描述「這個物品是否應該分到這個容器」。

採集模式中，採集 Prompt 用來描述「哪些方塊或資源類型應該被採集」。

建議先按「測試連線」確認 API 可用，再啟用 Prompt 分類或採集。

## 常見狀態與處理方式

| 狀態 | 可能原因 | 處理方式 |
| --- | --- | --- |
| 缺少燃料 | 燃料槽沒有有效燃料 | 在 GUI 左鍵燃料槽，自動從主手或背包放入燃料 |
| 尚未綁定來源銅箱 | 沒有設定來源銅箱 | 手持綁定板手右鍵銅箱 |
| 找不到可分類位置 | 來源物品沒有任何目的地可接收 | 加目標箱、調整接受/拒絕快取、或清出目標箱空間 |
| 缺少工具 | 採集模式沒有工具 | 停止後在 GUI 左鍵工具欄，自動從主手或背包放入工具 |
| 尚未設定作業區域 | 採集 Corner A 或 B 未完成 | 右鍵與 Shift+右鍵設定兩個角點 |
| 倉庫不可用 | 來源銅箱被破壞、未載入或滿了 | 修復來源銅箱、清空空間或重新綁定 |
| 工具損壞 | 採集工具耐久耗盡 | 停止後取回或替換工具 |

## 建議測試流程

分類模式：

1. 來源銅箱放 64 個石頭。
2. 目標箱放一個未滿 stack 的石頭。
3. 啟動後確認銅魁儡會搬 16 個以內並放入目標箱。
4. 把石頭移到拒絕快取，確認即使目標箱有石頭也不再放入。
5. 把石頭移回接受快取，清空目標箱，確認可放入空格。

採集模式：

1. 設定來源銅箱、兩個角點、工具與燃料。
2. 左鍵加入一種方塊作為採集目標。
3. 啟動後確認銅魁儡會照工具速度採集。
4. 確認採集倉庫未滿 16 個時會繼續找同類掉落。
5. 倉庫滿或找不到可合併目標時，確認會回來源銅箱存放。
