# 死亡背包與 `/back`

## 死亡背包流程

玩家死亡後，模組會記錄死亡前附近已存在的掉落物，再於死亡掉落完成後掃描死亡位置附近 10 格內新出現的物品實體。一般物品會被收進死亡背包；DeadRecall 的一般背包與死亡背包本身會保持獨立掉落。

1. 玩家死亡。
2. 模組等待原版完成物品掉落。
3. 收集死亡後新產生的一般掉落物。
4. 在死亡座標生成死亡背包。
5. 玩家等待 2 秒拾取延遲後，可拾起並右鍵開啟。

若物品欄為空，或附近沒有符合條件的新掉落物，不會生成死亡背包。

## 特性

| 屬性 | 行為 |
| --- | --- |
| 容量 | 依物品數動態計算，最多 54 格 |
| 搜尋範圍 | 死亡位置周圍 10 格 |
| 拾取延遲 | 40 ticks（2 秒） |
| 自然消失 | 不會自然消失 |
| 傷害 | 死亡背包本體免疫一般破壞 |
| 虛空 | 低於虛空傷害線時取消移除，給予向上動量並限制下墜 |
| 定位 | 地面上的死亡背包顯示紅色光柱 |
| 合併防護 | 每個死亡背包包含唯一資料，避免與其他死亡背包物品實體合併 |

## `/back`

模組會以玩家 UUID 暫存最近一次死亡的維度與座標。執行 `/back` 後，可從其他維度傳送回死亡地點；成功使用後會清除記錄，避免重複傳送。

## 實作安全條件

- 同一次死亡使用排程 guard，避免重複生成兩個死亡背包。
- 只收集死亡後新出現的 `ItemEntity`，避免誤收原本就在地上的物品。
- 背包介面固定追蹤開啟時的 ItemStack，避免關閉介面時把資料寫入後來換到手上的其他物品。
- 死亡背包清空後，會等介面關閉再移除背包物品，避免最後一格與滑鼠游標物品處理衝突。
